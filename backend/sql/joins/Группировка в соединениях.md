Более сложным вариантом использования соединений INNER/OUTER JOIN представляет их сочетание с выражениями группировки, в частности, с оператором GROUP BY. Например, выведем для каждого покупателя количество заказов, которые он сделал:

|   |   |
|---|---|
|1<br><br>2<br><br>3<br><br>4|`SELECT` `FirstName,` `COUNT``(Orders.Id)`<br><br>`FROM` `Customers` `JOIN` `Orders`<br><br>`ON` `Orders.CustomerId = Customers.Id`<br><br>`GROUP` `BY` `Customers.Id, Customers.FirstName;`|

![Группировка в INNER JOIN в PostgreSQL](https://metanit.com/sql/postgresql/pics/8.12.png)

Критерием группировки выступают Id и имя покупателя. Выражение SELECT выбирает имя покупателя и количество заказов, используя столбец Id из таблицы Orders.

Так как это INNER JOIN, то в группах будут только те покупатели, у которых есть заказы.

Если нужно получить также и тех покупателей, у которых нет заказов, то можно использовать OUTER JOIN:

|   |   |
|---|---|
|1<br><br>2<br><br>3<br><br>4|`SELECT` `FirstName,` `COUNT``(Orders.Id)`<br><br>`FROM` `Customers` `LEFT` `JOIN` `Orders`<br><br>`ON` `Orders.CustomerId = Customers.Id`<br><br>`GROUP` `BY` `Customers.Id, Customers.FirstName;`|

![Группировка в OUTER LEFT JOIN в PostgreSQL](https://metanit.com/sql/postgresql/pics/8.13.png)

Или выведем товары с общей суммой сделанных заказов:

|   |   |
|---|---|
|1<br><br>2<br><br>3<br><br>4<br><br>5|`SELECT` `Products.ProductName, Products.Company,`<br><br>        `SUM``(Orders.ProductCount * Orders.Price)` `AS` `TotalSum`<br><br>`FROM` `Products` `LEFT` `JOIN` `Orders`<br><br>`ON` `Orders.ProductId = Products.Id`<br><br>`GROUP` `BY` `Products.Id, Products.ProductName, Products.Company;`|
